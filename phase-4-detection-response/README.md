# Phase 4 — Detection and Response

This phase aggregates detection engineering output produced across all Phase 3 investigations. It serves as the centralized detection library — a single reference for all detection rules, Sigma rules, standing environmental patterns, ATT&CK coverage, and known audit gaps identified during live adversary simulation.

## Contents

- Detection rule index — all Splunk SPL rules produced across investigations with severity logic, validated event counts, and false positive exclusions documented
- Sigma rule index — all Sigma rules with detection branch documentation and ATT&CK mapping
- Standing notes registry — 23 confirmed environmental patterns accumulated across INV-001 through INV-005, applicable to every future investigation in this environment
- ATT&CK coverage map — techniques detected, detection sources, and coverage gaps per technique
- Audit policy configuration — confirmed active subcategories and known audit gaps with root cause documentation and compensating controls

## Key Principle

Every detection rule in this capstone was built from confirmed telemetry generated by live ART execution against a CIS-hardened Domain Controller. No rule is theoretical. Each is validated with documented field names, confirmed event counts, and known false positive patterns excluded with evidence.
