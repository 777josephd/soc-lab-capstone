## Detection Rule

**Alert name:** `[Persistence] T1053.005 - Suspicious Scheduled Task Creation`  
**File:** `splunk/T1053.005-scheduled-task-persistence.spl`  
**Schedule:** `*/5 * * * *`  
**Trigger:** Per result  
**Throttle:** 5 minutes on `Task_Name` field  
**Severity:** Dynamic per result — CRITICAL / HIGH / MEDIUM  
**Splunk indexes:** soc-lab-ad, sysmon-ad  
**Validated:** 2026-04-17 — 21 events, 1 CRITICAL, 18 HIGH, 2 MEDIUM

### Coverage

| EventCode | Source | Role | Detection target |
|-----------|--------|------|-----------------|
| 4698 | WinEventLog | Primary | Task creation — TaskContent severity tiering |
| 4688 | WinEventLog | Secondary | schtasks /create, Register-ScheduledTask, Invoke-CimMethod |
| Sysmon 1 | XmlWinEventLog | Secondary | Same as 4688 — compensates for known 4688 gaps |

### Severity Logic

```
CRITICAL — TaskContent contains IEX, FromBase64String,
            EncodedCommand, or GetItemProperty
            → Base64-encoded payload execution — Qakbot TTP
            → Triggers Ansible remediation via n8n

HIGH     — TaskContent contains powershell, cmd.exe,
            wscript, cscript, mshta, or rundll32
          — OR 4688/Sysmon 1 matching schtasks /create,
            Register-ScheduledTask, Invoke-CimMethod PS_ScheduledTask

MEDIUM   — All other 4698 events from non-machine,
            non-SYSTEM accounts
```

### False Positive Exclusions

| Pattern | Filter | Rationale |
|---------|--------|-----------|
| Machine accounts | `NOT Account_Name="*$"` | Task Scheduler housekeeping by WIN-hostname$ confirmed benign in INV-005 |
| SYSTEM account | `NOT Account_Name="SYSTEM"` | System-initiated task updates |
| 4702 task updates | Not included in rule | Machine account only — confirmed benign in INV-005 |

### Known False Positives

- Legitimate administrative task creation by authorized Domain Admins
- Software installation routines — typically run as SYSTEM (filtered)
- IT automation frameworks with identifiable management tool paths

## Sigma Rule

**File:** `sigma/T1053.005-scheduled-task-persistence.yml`  
**Status:** Experimental  
**Log source:** Windows process creation (product: windows, category: process_creation)  
**Level:** High

### Detection Branches

| Branch | Condition | Execution path |
|--------|-----------|----------------|
| selection_schtasks_cli | EventID 4688, NewProcessName endswith schtasks.exe, CommandLine contains /create | Test 1, Test 7 |
| selection_powershell_cmdlets | EventID 4688, CommandLine contains Register-ScheduledTask or New-ScheduledTaskAction | Test 4 |
| selection_wmi_cimmethod | EventID 4688, CommandLine contains Invoke-CimMethod AND PS_ScheduledTask | Test 6 |
| selection_encoded_execution | EventID 4688, CommandLine contains schtasks AND IEX | Test 7 encoded chain |

### Filters

- `SubjectUserName|endswith: '$'` — machine account exclusion
- `SubjectUserName: 'SYSTEM'` — system account exclusion

### Coverage Notes

This rule covers the process creation layer only. Full severity tiering requires
correlation with 4698 TaskContent inspection implemented in the Splunk rule.

The Tarrask hidden task variant is not covered here. Detection requires Sysmon
Event ID 12 monitoring for SD value deletion under TaskCache\Tree\[TaskName]\.
Reserved for a future T1564 investigation.

Notes confirmed during INV-005 that apply to future investigations of this
technique or related techniques.

| Note | Pattern |
|------|---------|
